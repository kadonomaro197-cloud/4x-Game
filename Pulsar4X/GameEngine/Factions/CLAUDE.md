## FactionDataStore тАФ the locked/unlocked split

`FactionInfoDB.Data` holds two parallel inventories:

- **`CargoGoods`** тАФ items available to the faction right now (mineable minerals, buildable materials, unlocked component designs).
- **`LockedCargoGoods`** тАФ items known to exist but not yet available (locked behind unresearched techs).

`Unlock(id)` moves an item from LockedCargoGoods to CargoGoods.
`IncrementTechLevel(tech)` increments the level, then calls `Unlock()` for each entry in `tech.Unlocks[tech.Level]`.

**IndustryDesigns sync gotcha:** `FactionInfoDB.IndustryDesigns` (the "things we can build" list) is populated at startup by `SetIndustryDesigns()`. When a `ProcessedMaterial` is unlocked mid-game via tech research, `Unlock()` moves it to `CargoGoods` but `IndustryDesigns` is NOT updated automatically тАФ so colonies can't queue the material even though it's now available. **Fix in place (`ResearchProcessor.cs`, this branch):** after `AddTechPoints` causes a level-up, `DoResearch()` iterates `tech.Unlocks[tech.Level]` and syncs any materials into `IndustryDesigns`.

## NPC Doctrine

`DoctrineVector` has four float weights. They are relative, not normalized тАФ whichever is largest drives that cycle's decision. Defaults to all zeros (no decision is made).

`NPCDecisionProcessor` registers via the standard `IHotloopProcessor` auto-discovery. It targets entities with `FactionInfoDB`. **The GlobalManager wiring gap is FIXED (keystone, 2026-06-30):** `MasterTimePulse.SimulateTimeUntil` now processes `GlobalManager.ManagerSubpulses` after the per-system block, so faction-level processors fire on their schedule (NPCDecisionProcessor: `FirstRunOffset` 5d, `RunFrequency` 30d). It still no-ops on player factions (`IsNPC == false`) and its `Tick` decision body is still a TODO тАФ but the *loop now runs*, which is the prerequisite every faction-level autonomous system (NPC turns, the internal/external politics engines) was waiting on. Liveness gauge: `NPCDecisionProcessor.TickCount` (static; climbs each ProcessManager call) тАФ asserted by `FactionEconomyTests.FactionLevelProcessors_FireOnceGlobalManagerIsIterated`.

**Phase 3.3 тАФ NPC treaty policy (the Ecosystem, 2026-07-12).** `NPCDecisionProcessor.RunTreatyPolicy(faction)` is the second live behaviour the monthly `Tick` can take (after `RunDiplomaticDrift`): for the first met, not-at-war rival whose relation score clears the NonAggression bar (`RelationshipState.HostileThreshold` = тИТ25), it calls `Treaties.Propose(faction, target, TreatyType.NonAggression, now)` тАФ turning the built-but-uncalled proposal machinery into live diplomacy (an NPC that actively seeks d├йtente, not just drifts). Gated by the new **`EnableDiplomaticProposals`** static (default false тАФ a *sibling* of `EnableOrderEmission` so combat/economy-order tests and diplomacy tests can be flipped independently). Guards: skip-if-`AtWar`, skip-if-already-`NonAggressionPact` (prevents monthly re-warm churn тАФ the #1 gotcha), skip self + `Game.NeutralFactionId`. One proposal per cycle. `Propose` signs BOTH ledgers, so the pact is two-sided. It auto-resolves through `WouldAccept` rather than surfacing an offer the player answers (a later refinement). **Four passes now (3.3 тЖТ 3.4 coalitions, 2026-07-12), each returns on first action (one treaty move per cycle):** **Pass 0 (3.4b тАФ join a pact-ally's war):** if a faction we hold a `DefensivePact` with is `AtWar` with X and we aren't, `Diplomacy.DeclareWar` on X (casus belli `AllyDefense`) тАФ two pacts vs a shared enemy become one COALITION whose guns actually fire (via the 3.4a combat war-override in `CombatEngagement.AreHostile`). **Pass C (3.4c тАФ the crack):** once the shared threat is gone (`ThreatAssessment.GreatestThreatTo == -1`), a LOW-Honour faction `Diplomacy.BreakTreaty`s the pact (gated by `Treaties.WouldKeepFaith` vs `PactAbandonTemptation` 0.5 тАФ high-Honour keeps faith); breaking re-arms the former allies. Mutually exclusive with Pass 1 (needs no threat vs needs a threat) тАФ "alliance forms vs a riser, cracks when it dies." **Pass 1 (3.2/3.3b sharpening):** if `GreatestThreatTo` names a feared rival, propose a **DefensivePact** to a TRUSTED (Allied-75) neighbour who isn't that threat. **Pass 2 (3.3):** the plain NonAggression d├йtente. Still byte-identical off (the whole method only runs when `EnableDiplomaticProposals` is on). Gauge: `NPCTreatyPolicyTests`.

## Gotchas

1. **`FactionDataStore` is not `FactionInfoDB`.** `FactionInfoDB.Data` is the DataStore. Code that gets the data store via `factionInfoDB.Data` and then calls `Data.CargoGoods[id]` is correct. Don't confuse the two.

2. **`IndustryDesigns` must be refreshed after any mid-game unlock.** The startup path (`SetIndustryDesigns`) and the startup-item path (`ColonyFactory.cs:43`) both do this correctly. Any new unlock call site must also update `IndustryDesigns` тАФ either by calling the sync loop or by calling `SetIndustryDesigns()`.

3. **NPCDecisionProcessor FIRES (GlobalManager wired, 2026-06-30) and its Tick now SETTLES a strategic objective (Movement II Phase 2.4b, 2026-07-11) тАФ but does not yet ACT on it.** The loop runs; `Tick` тЖТ `UpdateStrategicObjective` reads the needs-ladder (`NeedsLadder.AssessTier`), picks an objective (`ObjectiveSelector` over doctrine + `PersonalityDB`), commits it through the hysteresis engine (`ObjectiveTransition`), and stores a `StrategicObjectiveDB` on the NPC faction. This is the DECISION only тАФ the per-objective ORDER emission (GrowEconomyтЖТqueue industry, ExpandтЖТsurvey/colonize, ConquerтЖТbuild+attack) is the follow-on (2.4c+), so it's still byte-identical (a stored plan the brain hasn't acted on). Gauge `NPCObjectiveTickTests`.

4. **Every material a *starting* `ComponentDesign` needs must be in the colony blueprint's `StartingItems`.** `ColonyFactory.CreateFromBlueprint` unlocks `StartingItems` into `CargoGoods` first, then builds each `ComponentDesigns` entry тАФ and `ComponentDesigner` (`Engine/Components/ComponentDesigner.cs:63`) looks up every `ResourceCost` material in the **unlocked** `CargoGoods`. A required-but-not-unlocked material crashes New Game / Quickstart in `ComponentDesigner` (`GUID object {id} not found`). This happened when `electronics`/`ree-magnetics` were added to the starting laser weapon and to the Ship Yard / Research Lab installations (commit `28967c7`) without being added to `earth.json` `StartingItems` тАФ fixed by adding them. When you give a starting installation/weapon a new material cost, add that material to `StartingItems` too. (Related defensive change: `CargoDefinitionsLibrary.GetAny(string)` returns null instead of throwing `Sequence contains no elements`, so the failure names the missing id.) **Separate latent bug:** `TemplateFiles/ordnance.json` references `gallicite`, which is **not defined** as a mineral or material тАФ harmless until that ordnance design is built, then it faults the same way.
